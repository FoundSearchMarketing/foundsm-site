# Design System Checklist

- Use only approved `landingPage.sections` `_type` values.
- Use approved token values rather than arbitrary styling.
- Start with a hero and end with a CTA or form.
- Keep CTA language consistent and point to a real internal URL unless an external destination is intentional.
- Include useful `imageAlt` for meaningful media.
- Keep headings concise and avoid duplicate adjacent section ideas.
- Keep card grids to 2, 3, or 4 columns.
- Give feature tabs a stable lowercase `idPrefix` with letters, numbers, and hyphens.
- Reject raw CSS, inline styles, arbitrary colors, custom font sizes, custom font weights, custom breakpoints, and negative letter spacing.

If a page violates the checklist, revise it before final output and briefly note the correction.

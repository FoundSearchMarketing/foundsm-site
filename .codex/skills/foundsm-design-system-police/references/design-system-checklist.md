# Design System Checklist

- Page starts with a hero and ends with a CTA or form when reviewing full page JSON.
- All sections use approved `_type` values.
- Token fields use approved token values.
- No raw CSS, inline styles, arbitrary colors, custom font sizes, custom font weights, custom breakpoints, or negative letter spacing.
- CTA labels are consistent and CTA URLs are real paths.
- Meaningful media has useful alt text.
- Card grids use 2, 3, or 4 columns.
- Feature tabs use stable lowercase `idPrefix` values.
- Text should be concise enough to fit existing responsive components.

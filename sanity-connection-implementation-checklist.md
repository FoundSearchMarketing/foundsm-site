# Sanity Connection Checklist

Generated by comparing the current Astro routes in `src/pages`, Sanity schemas in `studio/schemas`, and Sanity data access in `src/lib`.

Summary:
- Sanity-backed public route groups: all current Astro public routes, including home page, About, Capabilities, capability detail pages, contact/newsletter forms, Our Approach, Team, Privacy Policy, 404, event landing page, insights index/blog/detail pages, and dynamic landing pages.
- Sanity schemas exist for blog posts, categories, authors, team members, landing pages, landing-page blocks, the home page, About, Capabilities, capability detail pages, and the remaining fixed page singletons.
- Current Astro public routes are editor-connected through Sanity where content is page-owned.
- `/lp/[slug].astro` is Sanity-backed, but it serves `/lp/*`; it is not equivalent to legacy root-level WordPress URLs unless redirects are added.
